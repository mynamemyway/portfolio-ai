# Детальный разбор классов в кодовой базе AI Portfolio bot

## **Основные классы и их функции:**

### **1. Класс `Settings` (app/config.py)**
**Назначение:** Управление настройками приложения через Pydantic
**Ключевые атрибуты:**
- `BOT_TOKEN`: Токен Telegram бота
- `OPENROUTER_API_KEY`: API ключ для доступа к LLM
- `OPENROUTER_CHAT_MODEL`: Основная модель LLM
- `OPENROUTER_FALLBACK_MODEL`: Резервная модель
- `SYSTEM_PROMPT`: Системный промпт для AI
- `MEMORY_WINDOW_SIZE`: Размер окна памяти диалога
- `SANITIZE_RESPONSE`: Флаг очистки MarkdownV2

**Важно:** Наследуется от `BaseSettings`, автоматическая загрузка из `.env`

---

### **2. Класс `ApiServiceEmbeddings` (app/core/rag.py)**
**Назначение:** Кастомный embedding сервис для LangChain
**Методы:**
- `aembed_documents()`: Асинхронная векторизация документов
- `embed_documents()`: Синхронная обертка для документов
- `aembed_query()`: Векторизация запроса
- `embed_query()`: Синхронная обертка для запроса

**Особенности:** Работает с внешним API эмбеддингов, поддерживает retry-логику

---

### **3. Класс `SQLiteChatMessageHistory` (app/core/memory.py)**
**Назначение:** Хранение истории диалогов в SQLite
**Методы:**
- `messages`: Получение сообщений сессии
- `add_message()`: Добавление одного сообщения
- `add_messages()`: Добавление пакетом
- `clear()`: Очистка истории

**Особенности:** Асинхронная работа, сериализация в JSON, совместим с LangChain

---

### **4. Класс `FallbackLoggingCallbackHandler` (app/core/chain.py)**
**Назначение:** Логирование переключений на резервную модель LLM
**Методы:**
- `on_llm_error()`: Обработка ошибок основного LLM

**Особенности:** Наследуется от `BaseCallbackHandler`, интегрируется в LangChain

---

### **5. Класс `MainMenuCallback` (app/keyboards.py)**
**Назначение:** Структурированные данные для callback-кнопок
**Атрибуты:**
- `action`: Действие кнопки
- `from_menu`: Откуда вызвано (опционально)

**Особенности:** Наследуется от `CallbackData`, автоматическая валидация

---

## **Ключевые функции и их логика:**

### **1. `get_rag_chain()` (app/core/chain.py)**
**Назначение:** Создание RAG-цепочки
**Логика:**
1. Инициализация основного и резервного LLM
2. Настройка векторного поиска
3. Создание промпт-шаблона
4. Форматирование документов
5. Сборка полной цепочки

**Особенности:** Поддерживает fallback-механизм, асинхронная работа

---

### **2. `process_query()` (app/handlers/user_handlers.py)**
**Назначение:** Обработка пользовательских запросов
**Логика:**
1. Отправка "typing" действия
2. Вызов RAG-цепочки
3. Обработка ответа (код-блок/Markdown/без форматирования)
4. Логирование статистики
5. Сохранение в историю

**Особенности:** Обработка ошибок, fallback для MarkdownV2

---

### **3. `create_vector_store()` (app/core/rag.py)**
**Назначение:** Создание векторной базы знаний
**Логика:**
1. Загрузка и разделение документов
2. Пакетная обработка для rate limiting
3. Векторизация и сохранение в ChromaDB

**Особенности:** Поддержка батчинга, обработка ошибок

---

### **4. `handle_main_menu_button()` (app/handlers/user_handlers.py)**
**Назначение:** Обработка нажатий на inline-кнопки
**Логика:**
1. Парсинг callback данных
2. Маршрутизация по action
3. Редактирование сообщения
4. Вызов RAG или изменение UI

**Особенности:** Поддержка вложенных меню, предопределенные вопросы

---

## **Архитектурные паттерны:**

### **1. Фабричные методы:**
- `get_embedding_model()`: Создание embedding модели
- `get_vector_store()`: Создание векторного хранилища
- `get_chat_memory()`: Создание объекта памяти

### **2. Обработчики событий:**
- `handle_start()`: Команда /start
- `handle_help()`: Команда /help
- `handle_reset()`: Команда /reset

### **3. Утилиты:**
- `sanitize_for_telegram_markdown()`: Очистка Markdown
- `escape_markdown_v2()`: Экранирование символов
- `log_query()`: Логирование статистики

## **Тестирование:**
Все ключевые компоненты покрыты модульными тестами с использованием:
- `pytest` + `pytest-asyncio`
- Полное мокирование внешних зависимостей
- Тестирование ошибок и edge cases

**Главный принцип:** Каждое действие пользователя проходит через обработчик → RAG-цепочка → обработка ответа → сохранение истории → логирование.
