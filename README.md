**AI Portfolio** — это интеллектуальный Telegram-бот, представляющий собой цифровую копию Python-разработчика. Бот предназначен для демонстрации профессиональных навыков, проектов и опыта специалиста в интерактивном формате. В основе бота лежит архитектура RAG (Retrieval-Augmented Generation), позволяющая давать точные и контекстуально-обоснованные ответы на основе приватной базы знаний.

## ✨ Ключевые возможности

*   **Интеллектуальные ответы (RAG):** Бот отвечает на вопросы, основываясь на документах из приватной базы знаний (`app/knowledge_base`), используя векторный поиск в **ChromaDB**.
*   **Расширенное интерактивное меню:** Реализована удобная навигация по разделам "Навыки", "Проекты", "Контакты" с помощью иерархических `inline`-меню.
*   **Быстрый доступ к информации:** Добавлены кнопки "About me", "Hard skills", "Soft skills" и "Tell about AI Portfolio" для получения ответов на предопределенные вопросы без необходимости вводить текст.
*   **Статические UI-элементы:** Команды `/help`, `/reset` и кнопка "Hello world!" работают без обращения к LLM, обеспечивая мгновенный отклик и улучшенный UX.
*   **Память диалогов:** Бот поддерживает контекст диалога в рамках одной сессии, что позволяет задавать уточняющие вопросы.
*   **Отказоустойчивость LLM:** Интеграция с **OpenRouter** и механизм фолбэка обеспечивают стабильную работу: при сбое основной модели система автоматически переключается на резервную.
*   **Гибкая конфигурация "личности" AI:** Системный промпт вынесен в переменные окружения, что позволяет изменять поведение и роль бота без изменения кода.
*   **Сбор статистики:** Реализован механизм сбора статистики по всем действиям пользователя. Отслеживаются популярные запросы и команды, что позволяет анализировать востребованность функционала.
*   **Модульная архитектура:** Код четко разделен на слои (`handlers`, `core`, `keyboards`, `utils`), что упрощает поддержку и расширение.
*   **Комплексное тестирование:** Проект полностью покрыт модульными тестами с использованием `pytest`, что гарантирует стабильность и качество кода.
*   **Готовность к контейнеризации:** Проект полностью готов к развертыванию с помощью Docker.

## ⚙️ Архитектурные компоненты

### 1. Ядро RAG (Retrieval-Augmented Generation)
Архитектура RAG является основой бота. Она позволяет генерировать ответы, основываясь не на общих знаниях модели, а на конкретных документах из приватной базы знаний.
- **Векторизация:** Для преобразования текста в векторы используется собственный API-сервис (`EMBEDDING_SERVICE_URL`), который работает на базе модели `intfloat/multilingual-e5-large`. Это обеспечивает полный контроль над процессом.
- **Векторное хранилище:** В качестве векторной базы данных используется **ChromaDB**. Она хранит векторы документов локально в директории `chroma_db/` и обеспечивает быстрый семантический поиск релевантного контекста.

### 2. Оркестрация LLM и отказоустойчивость
- **Оркестратор:** **LangChain** используется для построения и управления всей логической цепочкой (RAG-chain).
- **LLM-шлюз:** **OpenRouter** применяется как единая точка доступа к различным большим языковым моделям (LLM).
- **Механизм фолбэка:** Реализована система отказоустойчивости. Если основная модель (`OPENROUTER_CHAT_MODEL`) недоступна или возвращает ошибку, система автоматически переключается на запасную модель (`OPENROUTER_FALLBACK_MODEL`), обеспечивая стабильность работы.

### 3. Память диалога
- **Хранилище:** Для сохранения истории переписки с каждым пользователем используется база данных **SQLite** (`chat_history.sqlite3`).
- **Механизм:** Класс `SQLiteChatMessageHistory` обеспечивает асинхронный доступ к истории. `ConversationBufferWindowMemory` из LangChain ограничивает контекст последними `MEMORY_WINDOW_SIZE` сообщениями, что оптимизирует расход токенов.

### 4. Пользовательский интерфейс (UI)
- **Фреймворк:** **aiogram 3.x** используется для асинхронного взаимодействия с Telegram API.
- **Статические элементы:** Команды `/help`, `/reset` и кнопка "Hello world!" реализованы как статические, не требующие обращения к LLM. Это обеспечивает мгновенный отклик и улучшает UX.
- **Интерактивные меню:** Реализованы многоуровневые `inline`-клавиатуры для удобной навигации.
- **Поддержка медиа:** Реализована возможность прикреплять изображения к статичным сообщениям (`/start`, `/help`) через переменные окружения.

### 5. Санитайзер Markdown-разметки
- **Назначение:** В проекте используется кастомная функция `sanitize_for_telegram_markdown` для преобразования стандартного Markdown, генерируемого LLM, в формат, совместимый с `MarkdownV2` Telegram.
- **Логика:** Функция преобразует заголовки и жирный текст в единый стиль (`*текст*`), заменяет маркеры списков на `•` и экранирует специальные символы, которые могут вызвать ошибки парсинга в Telegram.

### 6. Сбор статистики и Тестирование
- **Сбор статистики:** Модуль `app/core/stats.py` отвечает за логирование всех действий пользователя (команды, нажатия кнопок, текстовые запросы) в таблицу `query_stats` базы данных SQLite. Это позволяет анализировать использование бота и эффективность RAG-системы.
- **Тестирование:** В директории `tests/` находится полный набор модульных тестов, написанных с использованием `pytest`. Тесты покрывают все ключевые компоненты приложения, включая обработчики `aiogram`, логику работы с базами данных и утилиты. Внешние зависимости (API Telegram, LLM) полностью мокируются, что обеспечивает изоляцию и надежность тестов.

## ⛓️ Логика обработки запроса (RAG-цепочка)

Весь процесс ответа на вопрос пользователя инкапсулирован в функции `process_query` и `get_rag_chain`.

1.  **Получение запроса:** Обработчик `handle_message` в `app/handlers/user_handlers.py` принимает текстовое сообщение пользователя.
2.  **Вызов цепочки:** Вызывается `process_query`, которая в свою очередь асинхронно запускает `conversational_rag_chain`.
3.  **Загрузка истории:** `_get_async_chat_history` извлекает последние сообщения из SQLite для текущей сессии.
4.  **Поиск контекста:** `retriever` (ChromaDB) находит в базе знаний фрагменты документов, семантически близкие к вопросу пользователя.
5.  **Формирование промпта:** `ChatPromptTemplate` собирает системный промпт, историю чата, вопрос пользователя и найденный контекст в единый запрос для LLM.
6.  **Генерация ответа:** LLM (с механизмом фолбэка) генерирует ответ.
7.  **Санитизация и отправка:** Ответ проходит через `sanitize_for_telegram_markdown` (если включено) и отправляется пользователю. В случае ошибки парсинга, отправляется исходный текст без форматирования (`parse_mode=None`).
8.  **Сохранение контекста:** Вопрос пользователя и ответ AI сохраняются в историю чата в SQLite.

## 📦 Инструкции по запуску

### Локальный запуск

1.  **Клонируйте репозиторий**
    ```bash
    git clone https://github.com/mynamemyway/ai-portfolio-bot.git
    cd ai-portfolio-bot
    ```

2.  **Создайте виртуальное окружение и установите зависимости**
    Убедитесь, что у вас установлен Python 3.12+.
    ```bash
    python3 -m venv venv
    source venv/bin/activate  # Для Linux/macOS
    pip install -r requirements.txt
    ```

3.  **Настройте переменные окружения**
    Создайте файл `.env` в корневой папке проекта.
    ```bash
    touch .env
    ```
    Откройте файл `.env` и добавьте в него следующие переменные, заменив значения на ваши:
    ```dotenv
    # Telegram Bot Token from @BotFather
    BOT_TOKEN="YOUR_TELEGRAM_BOT_TOKEN"

    # OpenRouter API Key from https://openrouter.ai/keys
    OPENROUTER_API_KEY="YOUR_OPENROUTER_API_KEY"

    # URL for the self-hosted embedding service API
    EMBEDDING_SERVICE_URL="http://your_server_ip:8000/embed"

    # (Optional) Controls the creativity of the response (e.g., 0.7).
    OPENROUTER_TEMPERATURE=0.7

    # (Optional) Limits the length of the generated response in tokens (e.g., 1024).
    OPENROUTER_MAX_TOKENS=1024

    # (Optional) The number of recent messages to keep in the conversation memory window.
    MEMORY_WINDOW_SIZE=10

    # (Optional) Path to the welcome photo for the /start command.
    WELCOME_PHOTO_PATH="path/to/your/welcome_photo.jpg"

    # (Optional) Path to the help photo for the /help command.
    HELP_PHOTO_PATH="path/to/your/help_photo.jpg"

    # (Optional) System prompt for the RAG chain.
    SYSTEM_PROMPT="Ты — русскоязычная цифровая копия..."
    ```

4.  **Запустите сервис эмбеддингов**
    Этот проект требует запущенного e5-large-embedding-service. Убедитесь, что Docker-контейнер с этим сервисом запущен и доступен по адресу, указанному в `EMBEDDING_SERVICE_URL`.

5.  **Наполните базу знаний**
    Поместите ваши документы (в формате `.md`) в директорию `app/knowledge_base/`.

6.  **Создайте векторный индекс**
    Запустите скрипт индексации. Он прочитает документы, векторизует их и сохранит в `chroma_db/`.
    ```bash
    python -m app.core.rag
    ```

7.  **(Опционально) Запустите тесты**
    Убедитесь, что вся внутренняя логика работает корректно.
    ```bash
    pytest
    ```

8.  **Запустите бота**
    ```bash
    python main.py
    ```

### Запуск с помощью Docker

Проект полностью сконфигурирован для развертывания с помощью Docker Compose, что значительно упрощает процесс.

1.  **Выполните шаги 1-5 из раздела "Локальный запуск"**:
    *   Клонируйте репозиторий.
    *   Создайте и настройте файл `.env`.
    *   Убедитесь, что сервис эмбеддингов запущен и доступен.
    *   Наполните директорию `app/knowledge_base` вашими документами.

2.  **Шаг 1: Создание векторного индекса**
    Перед первым запуском необходимо проиндексировать вашу базу знаний. Эта команда запустит временный контейнер, который выполнит скрипт индексации и завершит работу.
    ```bash
    docker-compose run --rm bot python -m app.core.rag
    ```

3.  **Шаг 2: Запуск бота**
    Запустите основной сервис в фоновом режиме. Docker Compose автоматически соберет образ (если он не был собран ранее) и запустит контейнер.
    ```bash
    docker-compose up -d
    ```
    *   Контейнер будет автоматически перезапускаться в случае сбоев благодаря политике `restart: unless-stopped`.
    *   Все необходимые тома для `ChromaDB` и `SQLite` уже настроены в `docker-compose.yml`.
    *   Для просмотра логов используйте команду: `docker logs -f ai-portfolio-bot`.
    *   Для остановки сервиса: `docker-compose down`.
